## 📐 Writeup Structure

Each writeup follows a consistent format:

```
challenges/
└── <category>/
    └── <challenge-name>/
        ├── README.md       ← Writeup (approach, steps, solution)
        └── solve.py        ← Exploit or solve script (where applicable)
```

Every `README.md` covers:

- **Challenge description** and source (platform, event)
- **Difficulty** and point value
- **Tools used**
- **Step-by-step approach** with reasoning
- **Key vulnerability or concept** explained
- **Flag** (redacted where required by platform rules)
